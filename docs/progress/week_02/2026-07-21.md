# Martes 21 de julio del 2026

## Objetido de la jornada

Preparar el Sistema Operativo base `Ubuntu 24.04 LTS` como servidor del proyecto.

## Actividades realizadas

- Paquetes generales de `Ubuntu`.
- Descarga de herramientas base del `SO` para el proyecto.
- Mantenimiento del `SO` en regla.
- Ingreso de `Docker` a `Ubuntu`.

## Problemas encontrados

- Los paquetes base de `Linux` no actualizados.
- Permisos de `Docker` no habilitados una vez instalarlo.
- Runtime `Nvidia` no registrado inicialmente en `Docker`.

## Soluciones

- Paquetes base de `Linux` actualizados y confirmados con `apt list --upgradable`.
- Permisos de administrador para `Docker` activados con `sudo usermod -aG docker $USER`.
- Configurado del runtime mediante `nvidia-ctk runtime configure --runtime=docker`.

## Criterios validados

### `Ubuntu 24.04 LTS` como Sistema Operativo Base

1. `apt update`.
2. `apt upgrade`.
3. `apt full-upgrade`
4. Sistema actualizado al 100% (`apt list --upgradable` en blanco).
5. Herramientas base instaladas.
6. `Python` operativo.
7. `Pip` activo.
8. `Git`operativo.
9. `Curl` operativo.

### `Docker` como contenedor

1. `Docker Engine 29.6.2` instalado.
2. `Docker Compose v5.3.1` instalado.
3. `Daemon` contesta correctamente.
4. El usuario ya pertenece al grupo `docker`.
5. Docker se ejecuta `Ubuntu 24.04 LTS` dentro de `WSL2`.
6. Se detectar 20 cores del procesador.
7. El kernel es de `WSL2`.
8. Sin errores de permisos.

### `NVIDIA Container Toolkit`

1. GPU detectada correctamente mediante `nvidia-smi`.
2. Acceso al dispositivo `/dev/dxg` confirmado.
3. Repositorio oficial registrado.
4. Toolkit instalado correctamente.
5. Runtime `nvidia` registrado en Docker.
6. `Docker` reconoce el runtime `nvidia`.
7. Contenedor `Ubuntu` accede correctamente a la GPU mediante `docker run --gpus all`.
